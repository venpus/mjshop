# 모바일/외부에서 테스트 서버 접속하기

같은 Wi‑Fi에 있는 스마트폰이나 다른 PC에서 개발 서버에 접속하는 방법입니다.

## 1. PC의 IP 주소 확인

- **Windows**: 명령 프롬프트에서 `ipconfig` 실행 후 **IPv4 주소** 확인 (예: `192.168.0.10`)
- **Mac**: 터미널에서 `ifconfig | grep "inet "` 또는 시스템 설정 → 네트워크에서 확인

## 2. 클라이언트(프론트) 설정

모바일 브라우저에서 **`http://PC_IP:5173`** 으로 접속하면, 앱이 **현재 주소의 호스트(PC IP)** 를 사용해 API를 호출합니다.  
따라서 **같은 Wi‑Fi 환경에서는 `client/.env.local` 없이도** 로그인 및 API가 동작합니다.

원하면 아래처럼 고정하고 싶을 때만 설정합니다 (예: PC IP가 `192.168.0.10`인 경우):

```env
VITE_API_URL=http://192.168.0.10:3000/api
VITE_SERVER_URL=http://192.168.0.10:3000
```

- `192.168.0.10`을 **본인 PC의 실제 IPv4 주소**로 바꿉니다.
- 설정 후 **클라이언트 개발 서버를 한 번 재시작**합니다.

## 3. 서버(백엔드) 설정

서버는 이미 `192.168.x.x` 출처를 CORS에서 허용하고 있습니다.  
다른 포트로 접속해도 되므로 **별도 설정 없이** 그대로 두면 됩니다.

필요하다면 **`server/.env`**에서:

```env
CLIENT_URL=http://localhost:5173,http://192.168.0.10:5173
```

처럼 PC IP 기준 클라이언트 URL을 추가할 수 있습니다 (이미 정규식으로 로컬 IP를 허용 중이면 생략 가능).

## 4. 서버·클라이언트 실행

1. **서버**: `server` 폴더에서 `npm run dev` (포트 3000, 모든 인터페이스에서 수신)
2. **클라이언트**: `client` 폴더에서 `npm run dev` (포트 5173, 네트워크 접속 허용)

실행 시 터미널에 예시처럼 표시됩니다:

- Vite: `Local: http://localhost:5173/` 와 `Network: http://192.168.0.10:5173/`
- 서버: `서버가 포트 3000에서 실행 중입니다.`

## 5. 모바일에서 접속

스마트폰을 **같은 Wi‑Fi**에 연결한 뒤, 브라우저 주소창에 다음처럼 입력합니다:

```
http://192.168.0.10:5173
```

(위 `192.168.0.10`을 2번에서 사용한 PC IP로 바꿉니다.)

- 첫 화면·로그인·API 호출이 모두 이 PC의 서버/클라이언트로 동작해야 합니다.
- **데이터 요금(모바일 데이터)** 이 아닌 **Wi‑Fi**로 접속해야 같은 네트워크로 인식됩니다.

## 6. 접속이 안 될 때

- **PC 방화벽**: Windows 방화벽에서 포트 3000, 5173 인바운드 허용
- **동일 네트워크**: PC와 휴대폰이 같은 Wi‑Fi(같은 공유기)에 연결되어 있는지 확인
- **주소 확인**: 모바일에서 `http://PC_IP:5173` 사용 여부, `.env.local`의 `VITE_API_URL`/`VITE_SERVER_URL`에 같은 PC IP가 들어갔는지 확인

## 7. 다른 네트워크(집 밖)에서 접속하려면

같은 Wi‑Fi가 아닌 **외부(4G/5G, 다른 공유기)** 에서 접속하려면 터널 도구가 필요합니다.

- **ngrok**: `ngrok http 5173` 으로 공개 URL 발급 후, 해당 URL을 모바일에서 접속
- **Cloudflare Tunnel** 등으로 PC의 5173/3000 포트를 인터넷에 노출

이 경우 발급된 URL을 기준으로 `VITE_API_URL`/`VITE_SERVER_URL`을 설정하고, 보안(인증·HTTPS)을 함께 고려하는 것이 좋습니다.
