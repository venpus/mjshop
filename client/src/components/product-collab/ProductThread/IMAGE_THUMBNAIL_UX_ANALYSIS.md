# 대표 이미지 썸네일 액션 UI/UX 분석

## 현재 구조의 문제

- **썸네일당 두 개의 오버레이 버튼**
  1. **전체 영역**: "대표로 설정" (absolute inset-0, 호버 시 반투명 검정 + 텍스트)
  2. **우측 상단**: 삭제 버튼 (absolute top-right, 휴지통 아이콘, 호버 시 표시)

### 모바일에서 겹치는 이유

1. **호버 부재**: 터치 기기에는 `:hover`가 없거나, 탭 한 번으로 호버 상태가 고정된다. 한 번 탭하면 두 버튼이 동시에 보일 수 있음.
2. **작은 터치 영역**: 썸네일이 80×80px로 작아서 "대표로 설정" 영역과 "삭제" 버튼이 사실상 겹친다. 삭제를 누르려다 대표 설정이 실행되거나 그 반대가 발생하기 쉽다.
3. **의도 불명확**: 같은 카드 위에 두 액션이 겹쳐 있어, 사용자가 어떤 동작이 나올지 예측하기 어렵다.

## 개선 방향 후보

| 방안 | 설명 | 장점 | 단점 |
|------|------|------|------|
| **A. 썸네일별 ⋮ 메뉴** | 카드마다 "더보기(⋮)" 버튼 하나만 두고, 탭 시 "대표로 설정" / "삭제" 메뉴 표시 | 터치/마우스 동일 동작, 영역 겹침 없음, 패턴 익숙함 | 메뉴를 한 번 더 열어야 함 |
| **B. 썸네일 아래 고정 액션 바** | 각 썸네일 아래에 [대표로 설정] [삭제] 버튼을 항상 표시 | 항상 보여서 발견 용이 | 세로 공간 많이 사용, 썸네일 많을 때 지저분 |
| **C. 롱프레스 컨텍스트 메뉴** | 썸네일 길게 누르면 메뉴 표시, 짧게 누르면 확대 등 | 카드 자체는 깔끔 | 롱프레스 학습 필요, 접근성/발견성 낮음 |
| **D. 탭 시 액션 시트(모바일)** | 썸네일 탭 시 하단 시트로 "대표로 설정" / "삭제" 제시 | 모바일 친화적 | 반응형 분기 필요, 데스크톱과 패턴 불일치 |

---

## 권장안: **썸네일별 ⋮ 메뉴 (방안 A)**

- **한 개의 명확한 터치 타깃**: 썸네일당 "⋮(MoreVertical)" 버튼 하나만 두고, 탭/클릭 시 드롭다운으로 "대표로 설정", "삭제"를 노출.
- **모바일·데스크톱 동일**: 호버에 의존하지 않고 클릭/탭으로만 동작하므로 모바일에서도 대표 설정과 삭제가 겹치지 않음.
- **이미 프로젝트에 있는 DropdownMenu 활용**: Radix 기반 드롭다운으로 접근성·키보드·포커스 처리 가능.

### 적용 시 기대 효과

- 삭제와 대표 설정이 서로 다른 메뉴 항목으로 분리되어 잘못된 동작 감소.
- 작은 썸네일에서도 "⋮"만 누르면 되므로 터치 정확도 요구가 낮아짐.
- UI 패턴이 일반적이라 추가 설명 없이도 이해하기 쉬움.
